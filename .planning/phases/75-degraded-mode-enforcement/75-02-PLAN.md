## Locked Decisions

Decision: truth-bearing planning and verification workflows must block when degraded policy evaluates to `UNSAFE`.
Decision: model-facing memory includes planner context build and canonical executor writeback when presented as trusted workflow memory.
Decision: planning flows do not get partial trust; they are memory-backed and trustworthy or memory-disabled.
Decision: missing or stale drift or reconciliation truth can make a workflow `UNSAFE` if that workflow claims current truth posture.
Decision: top-level CLI gating and subsystem-level fail-closed behavior are both mandatory.
Decision: blocked flows must emit structured failure classifications with subsystem, canonical state, reason, implications, and next options.

## Risks

Risk: route-level blocks can miss deep call paths and leave unsafe truth-bearing work partially runnable.
Guardrail: keep top-level CLI gating and subsystem-level backstops both in place, with tests covering each layer.
Fallback: if a workflow cannot prove it remains trustworthy under degraded conditions, block it rather than downgrading silently.

## Assumptions

Assumption: Phase 75 can reuse one degraded-policy module to make both reader and enforcement decisions, avoiding a second policy vocabulary for blocked flows.
Assumption: diagnostic surfaces such as health and status remain allowed even while planning and truth-bearing verification routes are blocked.
Deferred: exact list of gated verification subcommands may expand during implementation so long as all current-truth-claiming paths are covered by phase closeout.
