# PLAN: 09-01 - Add adversarial ambiguity and lockability harness to ITL

## Goal
Extend the ITL ambiguity layer with an adversarial pass that distinguishes normal clarification needs from invariant lockability, so inferred constraints can later be gated on a test-backed `lockable` result instead of planner prose alone.
