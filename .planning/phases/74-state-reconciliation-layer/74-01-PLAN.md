## Locked Decisions

Decision: Phase 74 consumes the Phase 73 drift report and applies consequences rather than recomputing drift.
Decision: Reconciliation may mutate sanctioned truth surfaces, but it must not invent evidence or rewrite history.
Decision: Historical-only drift is visible but non-blocking for current-state downgrade.
Decision: Re-verification requirements must be machine-readable and attributable to the drift report that triggered them.
Decision: Reconciliation must emit a central machine artifact that records source-report identity, applied changes, unchanged surfaces, and attribution metadata.

## Risks

Risk: reconciliation can mutate too broadly and create a second source of truth.
Guardrail: mutations must flow through one deterministic reconciliation layer and sanctioned state/roadmap interfaces.
Fallback: if a downgrade target is ambiguous, record a re-verification-required or conditional outcome rather than silently mutating an uncertain surface.

## Assumptions

Assumption: Phase 74 consumes the Phase 73 report fields `affected` and `predicted_effect` as reconciliation inputs rather than recomputing detection logic.
Deferred: if the Phase 73 report shape changes during implementation, update the reconciliation adapter layer rather than broadening Phase 74 back into a second detection engine.
