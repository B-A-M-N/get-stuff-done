## Locked Decisions

Decision: Worklist comes exclusively from 81-01's phase-closure-diff.json. No hard-coded artifact lists.
Decision: For each item:
  - If status_before was MISSING: CREATE (generate fresh)
  - If status_before was STALE/INVALID: REPAIR (generate to temp, compare, replace if different)
Decision: After generation, run immediate validator. If does not achieve VALID, mark as escalations and HALT.
Decision: 81-02 is responsible for generating:
  - missing VERIFICATION.md for any phase
  - missing TRUTH.yaml for any phase
  - missing VALIDATION.md for any phase
  - repair of invalid VERIFICATION only (TRUTH/VALIDATION regenerated after VERIFICATION fixes)
Decision: Do NOT create self-artifacts for Phase 81 in this plan; those belong to 81-03 after closure is proven.
Decision: All generator commands must be idempotent and produce deterministic output.

## Risks

Risk: Regenerating TRUTH/VALIDATION might overwrite manual edits.
Guardrail: Only regenerate if inventoried as MISSING/INVALID. Skip if VALID_EXISTING.
Fallback: if manual edits detected (non-deterministic generation), preserve and flag for human review.

Risk: Inventory might be stale if files changed during 81-01.
Guardrail: re-verify file existence immediately before generation; if state changed, re-run inventory or skip.

Risk: Generator commands could fail due to invalid source VERIFICATION.
Guardrail: run `gsd-tools verify verification-artifact` on source first; if invalid, escalate and halt.

## Assumptions

Assumption: 81-01 produced a complete and accurate inventory.
Assumption: All necessary source material (summaries, verification templates) is present to generate missing artifacts.
Assumption: phase-truth generation is deterministic and safe to rerun.
