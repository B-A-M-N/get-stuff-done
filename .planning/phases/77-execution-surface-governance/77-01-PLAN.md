## Locked Decisions

Decision: command governance is centralized in `.planning/policy/command-governance.yaml`.
Decision: classes are `ungated_execution`, `warn_only`, `hard_gated_state_transition`, and `recovery_only`.
Decision: classification is per command plus subcommand plus mode.
Decision: new commands default to `warn_only`.
Decision: Phase 77 reuses the Phase 75 degraded-state artifact rather than adding a second governance artifact.

## Risks

Risk: governance logic can drift if YAML policy and route lookup do not stay aligned.
Guardrail: one helper module should own policy loading, normalization, and route lookup, backed by focused tests.
Fallback: if a route cannot be classified, default to `warn_only` rather than silently ungated execution.

## Assumptions

Assumption: the policy map can be represented in YAML without introducing new dependencies beyond existing repo parsing patterns.
Assumption: route keys can be normalized deterministically enough that later CLI wiring remains readable.
Deferred: exact recovery-only command inventory may be expanded during implementation if needed, but the core recovery set is already locked.
