# PLAN: 02-02 - Integrate ITL into dostuff and add adversarial coverage

## Goal
Upgrade `dostuff` from a simple router into the first narrative-first ITL workflow that previews a structured interpretation, handles ambiguity safely, writes an audit record, and only then routes to the next GSD command.

- `get-stuff-done/workflows/dostuff.md`
- `tests/dostuff.test.cjs`

## Tasks
<task type="auto">
  <name>Upgrade dostuff to use the ITL core</name>
  <files>commands/gsd/dostuff.md, get-stuff-done/workflows/dostuff.md, get-stuff-done/bin/lib/*.cjs</files>
  <action>

## Verification
- `dostuff` can display an interpretation summary for narrative input.
- Ambiguous narratives trigger clarification-safe behavior instead of silent routing.
- Audit entries include raw narrative, normalized interpretation, ambiguity findings, and assumptions.
