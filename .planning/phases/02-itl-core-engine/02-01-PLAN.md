# PLAN: 02-01 - Build ITL core primitives and audit persistence

## Goal
Create the internal Intent Translation Layer modules that normalize extracted narrative intent, score ambiguity, render interpretation summaries, and persist audit data in a local SQLite-backed store.

## Read First
- `commands/gsd/dostuff.md`
- `get-stuff-done/workflows/dostuff.md`
- `get-stuff-done/bin/gsd-tools.cjs`
- `get-stuff-done/bin/lib/frontmatter.cjs`
- `get-stuff-done/bin/lib/verify.cjs`

## Tasks
<task type="auto">
  <name>Create ITL module surface</name>
  <files>get-stuff-done/bin/lib/itl-*.cjs</files>
  <action>
    Add a small internal module set for:
    1. intent normalization / validation,
    2. ambiguity scoring,
    3. summary formatting,
    4. audit persistence.
    Keep the interface provider-agnostic and compatible with existing CommonJS modules.
  </action>
  <acceptance_criteria>
    - Narrative interpretation can be represented as a single normalized object.
    - Ambiguity findings include at least issue type, message, and confidence/severity data.
    - Summary rendering is deterministic for the same normalized input.
  </acceptance_criteria>
  <verify>Module-level tests cover normalization, ambiguity scoring, and summary output.</verify>
</task>

<task type="auto">
  <name>Add SQLite-backed audit trail</name>
  <files>get-stuff-done/bin/lib/itl-audit.cjs, package.json, tests/*.test.cjs</files>
  <action>
    Implement local persistence for raw narrative input, normalized interpretation output, ambiguity findings, and explicit assumptions under `.planning/itl/`.
    Keep schema creation idempotent and safe for repeated command runs.
  </action>
  <acceptance_criteria>
    - Audit storage is created automatically when missing.
    - Each interpretation run records raw input and normalized output together.
    - Repeated runs do not corrupt or duplicate schema setup.
  </acceptance_criteria>
  <verify>Tests prove schema initialization, write, and readback behavior.</verify>
</task>

- The normalized intent object captures goals, constraints, preferences, anti-requirements, success criteria, risks, and unknowns.
