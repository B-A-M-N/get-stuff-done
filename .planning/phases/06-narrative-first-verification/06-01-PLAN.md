# PLAN: 06-01 - Add ITL-backed narrative intake to source verify-work workflow

## Goal
Add a narrative-first entry stage to the canonical source `verify-work` command/workflow so verification starts from interpreted user outcomes and success criteria before the UAT loop begins, while preserving test extraction, issue logging, severity inference, diagnosis handoff, and UAT.md output.
