## Locked Decisions

Decision: Phase 79 is a hard release gate for milestone `v0.7.0`, not a soft validation pass.
Decision: the authoritative proof surface is real CLI behavior plus emitted truth artifacts, not helper-only tests or mock-only simulations.
Decision: the gauntlet must cover the full truth chain from execution through commit proof, verification, memory truth, retrieval-facing posture, and governance.
Decision: failure injection is limited to realistic artifact corruption, CLI misuse, subsystem degradation, and state-drift contradiction; it may not bypass the enforcement boundary to fabricate impossible states.
Decision: every scenario resolves to exactly one of `INVALID`, `CONDITIONAL`, `RECONCILIATION_REQUIRED`, or `BLOCK`.
Decision: the plan must deliver the mandatory artifact family: `79-GAUNTLET-SPEC.md`, `79-GAUNTLET-RESULTS.md`, `79-COVERAGE-MAP.md`, `79-DRIFT-REPORT.md`, and `79-VERIFICATION.md`.
Decision: deterministic local mode is the release-gating source of truth and assumes a Node runtime with `node:sqlite` support because the repo’s canonical fallback memory and audit paths depend on it.
Decision: live integration mode is optional and capability-gated, but when available it must preserve the same truth classification semantics as deterministic mode.
Decision: the scenario catalog must include explicit hostile coverage for authoritative context-build / Firecrawl posture, memory-truth contradictions, retrieval-facing truth posture, Plane-configured truth paths when available, declared degradation, and undeclared degradation.
Decision: milestone completion must consume the Phase 79 verification verdict so a failing gauntlet truly blocks `v0.7.0` closeout.

## Risks

Risk: the gauntlet could drift into a second truth engine if it classifies outcomes using custom logic instead of observed CLI behavior and canonical artifacts.
Guardrail: use helpers only to seed corruption and normalize results; final classification must derive from subprocess outcomes and authoritative artifacts such as verification, phase truth, degraded-state, and drift surfaces.
Fallback: if a scenario cannot be expressed through sanctioned CLI surfaces without inventing impossible state, drop or redesign the scenario rather than weakening the enforcement boundary.

Risk: deterministic local mode could silently run under an older Node runtime that lacks `node:sqlite`, invalidating memory-fallback and audit-path scenarios.
Guardrail: gauntlet startup must fail fast when the runtime cannot provide `node:sqlite`.
Fallback: live-only or memory-fallback scenarios are marked unavailable only when their absence is explicitly surfaced and the release gate remains blocked until the runtime prerequisite is satisfied.

Risk: scenario sprawl could produce a large catalog without clear requirement traceability or a trustworthy final verdict.
Guardrail: maintain one scenario catalog, one coverage map, and one result normalization shape consumed by final verification.
Fallback: if artifact generation becomes too coupled, keep raw run ledgers machine-readable and render markdown summaries from those ledgers.

Risk: the phase could produce a truthful failing verdict that milestone completion ignores.
Guardrail: wire Phase 79 verification into the milestone-closeout path or equivalent authoritative completion gate.
Fallback: if milestone completion cannot consume the artifact automatically yet, completion must fail closed until the verifier is explicitly checked in the closeout path.

## Assumptions

Assumption: one plan is enough if it separates harness foundation, adversarial scenario implementation, and release-gate verification closeout.
Assumption: existing fixture and CLI helper patterns in `tests/helpers.cjs`, `tests/degraded-mode-enforcement.test.cjs`, `tests/command-governance-enforcement.test.cjs`, and `get-stuff-done/bin/lib/enforcement-boundary-audit.cjs` are sufficient starting points for deterministic project setup and corruption seeding.
Assumption: live-mode Plane, Firecrawl, and Postgres checks can be capability-gated without weakening the deterministic release gate.
Deferred: correcting `package.json` engine metadata to match the repo’s real runtime floor is adjacent repo hygiene, not required for Phase 79 plan completion.
