  <action>
    For each agent .md file (all that load context):
    1. Identify sections where the agent currently loads context: typically using `Read` tool for .planning/ files and `WebSearch`/`WebFetch` for external documentation.
    2. Replace those with a unified block:
       - Check Firecrawl availability:
         ```bash
         FC=$(node "$HOME/.claude/get-stuff-done/bin/gsd-tools.cjs" firecrawl check 2>/dev/null)
         FIRECRAWL_UP=$(echo "$FC" | node -e "try{const d=JSON.parse(require('fs').readFileSync('/dev/stdin','utf8'));process.stdout.write(d.available?'yes':'no')}catch{process.stdout.write('no')}")
         if [[ "$FIRECRAWL_UP" != "yes" ]]; then echo 'Firecrawl unavailable'; exit 1; fi
         ```
       - Build unified spec JSON with shell or node. Include all necessary sources for that agent (e.g., .planning/STATE.md, .planning/ROADMAP.md, .planning/research/*.md, external URLs if needed). Use a here-doc or node to construct JSON:
         ```bash
         SPEC=$(node -e "console.log(JSON.stringify({sources:['.planning/STATE.md','.planning/ROADMAP.md','https://docs.example.com'], options:{normalize:true}}))"
         ```
       - Submit:
         ```bash
         RESPONSE=$(curl -s -X POST http://localhost:3002/v1/context/crawl -H "Content-Type: application/json" -d "$SPEC")
         ```
       - Parse response: extract `artifacts` array; for each artifact, either write `content_markdown` to a temporary file or inject directly into the model's context via a tool. For simplicity, agents can use `Write` tool to create a temporary summary file and then `Read` it, or directly embed in the narrative if the tool allows. Document approach.
    3. Remove any `Read` tool calls for files now delivered via the spec.
    4. Remove any `WebSearch` or `WebFetch` calls for external docs; those URLs go into the spec sources instead.
    5. Ensure performance: set `max_total_bytes: 10485760` and `timeout_ms: 30000` in options.
    6. Update the agent's tool declarations if needed (Bash/curl usually already allowed).
    7. Add comments explaining the new pattern.
    The updated agent should produce the same functionality but through the unified Firecrawl API.
  </action>
