## Locked Decisions

Decision: the authoritative artifact is `N-TRUTH.yaml`; `N-TRUTH.md` is a rendered human-readable view.
Decision: phase truth is a new synthesis contract and must not extend or replace `VERIFICATION.md`.
Decision: generation is explicit through `phase-truth generate <phase>`, even if workflows also auto-trigger it.
Decision: final status is derived with the Phase 78 precedence model: reconciliation, then drift, then verification, then summary/proof, then degraded-state caveats.
Decision: validators are required now; authority signing is deferred.
Decision: consumed verification artifacts must be validated against the existing evidence-first verification contract before phase truth treats them as trustworthy inputs.
Decision: workflow-triggered regeneration is required after verification completion, reconciliation affecting a phase, and phase completion.

## Risks

Risk: phase truth can drift into a second verifier if it re-parses low-level evidence too aggressively or duplicates verification semantics.
Guardrail: keep the new artifact focused on synthesis, source references, explicit gaps, and final status derivation; do not copy whole verification tables or proof blocks into the new contract.
Fallback: if backfill inputs are incomplete, emit gap records and `CONDITIONAL` or `INVALID` rather than fabricating a full `VALID` result.

## Assumptions

Assumption: current drift and reconciliation artifacts can be filtered to phase-affected entries using existing fields or conservative matching.
Assumption: markdown rendering can be generated from the machine object without adding a new template dependency.
Assumption: the existing sanctioned commands for verification, reconciliation, and phase completion are the right trigger points for automatic regeneration hooks.
Deferred: milestone-level truth aggregation and historical backfill live in follow-up plan 78-02.
