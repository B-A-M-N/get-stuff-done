## 🔑 Key Decisions

1. **Query validation BEFORE execution** — intent tier classification and boundary checks in `validateQueryBoundary()` prevent full scans and Tier 3-only queries.
2. **Depth limit 2** — provenance graph stops at artifact → section → atoms/citations; no cross-mission recursion.
3. **Node cap 1000** — enforced upfront (estimated) and post-construction; exceeds → error returned, no truncation.
4. **Strict equivalence for replay** — string equality after timestamp normalization; any mismatch = drift (CONTENT_MISMATCH).
5. **Metrics advisory only** — explicitly documented; metrics must never modify truth or influence gates.
6. **GIN index optional** — added but not required for small datasets; plan notes it's for scale.
7. **findSimilarArtifacts scoping** — limited to same mission (practical); plan's "all missions" would violate query boundaries; noted as potential future expansion.

---

## ⚠️ Blockers / Risks

1. **YAML parsing dependency** — replay uses direct string comparison after timestamp override. If `derivePhaseTruth` output formatting changes (non-deterministic ordering), replay may false-positive drift. This is acceptable (captures drift) but may need canonicalization later (js-yaml).
2. **Replay integration unverified** — not yet run against real Phase 52 artifact in live DB. Should test: `replayArtifact('syn_7aa51411a4c1de59')` returns `matches: true`.
3. **findSimilarArtifacts mission scope** — current implementation uses same mission for safety; plan suggested all missions. This requires either boundary relaxation or iteration over known missions. Deferring to later phase (14?).
4. **Test concurrency issues** — running `npm test` fails for unrelated older tests. Phase 12 unit tests pass in isolation. Full suite health needs later sweep.
